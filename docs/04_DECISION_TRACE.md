# Decision Trace (Why these rules)

본 문서는 “왜 이런 룰을 채택했는가”를 **현재 코드에 맞춰** 간단히 남깁니다.

## 1) 중립 점령 = 즉시
- 요구: 중립 점령에 확률/민병대 개입 금지
- 반영: owner가 `NEUTRAL`이면 전투 없이 즉시 owner 변경

## 2) 점령 성공 시 유닛 이동
- 요구: 유닛이 “턴마다 이동”하는 전략 시뮬 감각
- 반영: 점령 성공 시 공격 유닛을 대상 지역으로 이동 처리

## 3) BASE_GARRISON = 60
- 의도: 유닛이 없는 점령지도 너무 허무하게 뚫리지 않도록 최소 방어력을 부여
- 범위: **점령지에만 적용**, 중립에는 적용하지 않음

## 4) 전투 승리 임계값 = 2.0
- 요구: 공수비 1:1이면 수비 성공, 공격이 2배면 점령
- 반영: `ratio = Attack / EffectiveDefense`로 계산 후 `ratio >= 2.0`이면 점령

## 5) WALL 방어 보정 = +40%
- 요구: 성벽이 30%~50% 수준에서 방어 보정
- 반영: 현재는 40%로 고정

## 6) 상륙 실패는 유닛 파괴(간소)
- 의도: 상륙 실패 처리(되돌리기/포로/퇴각 등)는 복잡도가 커지므로, 현재는 단순화
